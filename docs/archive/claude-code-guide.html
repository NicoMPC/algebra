01 — Slash Commands

Les commandes à connaître

/compact
Compresse le contexte quand la session devient longue, sans tout perdre. Garde la conscience de la tâche. → Utiliser quand le contexte grossit sur une même tâche.
/clear alias /new
Reset complet. Libère tout le contexte. Idéal entre deux tâches différentes. → /compact garde la mémoire, /clear repart de zéro.
/rename <nom>
Nomme la session courante. Essentiel pour retrouver tes sessions plus tard. → Ex: /rename agents-orchestration
/fork
Crée une copie exacte de la conversation à cet instant T. Explore une idée sans risque. → L'équivalent d'un git branch pour ta conversation.
/resume alias /continue
Reprend une session passée. Sans argument = ouvre un picker. Depuis le CLI : claude -c pour la plus récente. → Gérer une urgence, puis revenir exactement où tu en étais.
/diff
Ouvre un viewer interactif. Flèches ←→ pour basculer entre git diff et les diffs par tour. ↑↓ pour naviguer entre fichiers. → Toujours faire avant de committer après un gros refacto.
/plan
Mode planification. Claude propose un plan structuré avant d'exécuter quoi que ce soit. → Pour les chantiers complexes : archi, nouveaux agents, gros refacto.
/model
Change de modèle à la volée. Haiku = rapide/léger. Sonnet = défaut. Opus = décisions complexes. → Economise des tokens en switchant sur Haiku pour les tâches simples.
/cost
Affiche la consommation de tokens de la session en cours. → Bonne habitude à prendre, surtout avec Opus.
/tasks + /kill <id>
Liste les tâches qui tournent en arrière-plan. /kill <id> pour en stopper une. → Préfixe tes commandes avec & pour les lancer en background.
/btw <question>
Pose une question rapide sans polluer le contexte principal de ta tâche. → "En passant, comment fonctionne UrlFetchApp ?"
/release-notes
Affiche les nouveautés de ta version courante. La meilleure façon de rester à jour.
02 — Raccourcis Clavier

Les gestes à automatiser

Échap
Stoppe Claude sans quitter (Ctrl+C quitte tout)
Échap × 2
Liste de tous les messages précédents pour y revenir
Shift + Tab
Cycle entre Auto-Accept, Plan Mode et mode normal
↑
Navigue dans l'historique des chats (même des sessions passées)
Ctrl + V
Coller une image (pas Cmd+V sur Mac — oui c'est contre-intuitif)
Drag & Drop
Glisse une image dans la fenêtre pour l'analyser ou l'implémenter
03 — Scénario Pratique

Gérer une interruption sans perdre le fil

terminal — session active
1
Tu bosses sur tes agents. Un bug urgent remonte sur Matheux.
Nomme ta session : /rename agents-orchestration
2
Ouvre une nouvelle session Claude Code dans un autre terminal (ou onglet). Lance /clear pour contexte propre, gère ton bug.
3
Bug réglé. Dans n'importe quelle session, tape /resume. Le picker affiche toutes tes sessions nommées — sélectionne agents-orchestration.
4
Tu reprends exactement là où tu t'étais arrêté. Contexte intact, aucune perte.
04 — Scénario Pratique

Tester une idée sans risquer la session principale

terminal — fork workflow
1
Claude a bien compris ton contexte. Tu as une idée d'approche différente mais t'es pas sûr.
2
Tape /fork. Claude crée une copie exacte de la conversation à cet instant T.
3
Si ça te plaît pas : ferme cette branche, /resume → session originale intacte.
Si ça marche : continue sur cette branche, elle devient ta session principale.
05 — Tricks Avancés

Les patterns qui changent la qualité

ultrathink — Le mot magique à glisser dans n'importe quel prompt pour activer le raisonnement étendu de Claude. Idéal pour les décisions d'archi, les bugs complexes, la planification d'agents.

Exemple : "Ultrathink et propose-moi 3 architectures pour mon système d'agents plan/exec"

! <commande>
Exécute une commande shell directement et injecte son output dans le contexte, sans que Claude l'interprète. → ! git status — Claude voit le résultat, mais ne gère pas la commande.
& <commande>
Lance une tâche en arrière-plan pendant que tu continues à bosser. → & npm run dev — serveur lancé, Claude continue.
@fichier
Référence un fichier spécifique dans le contexte sans qu'il soit ouvert. → "Regarde @backend.js et dis-moi si la logique boost est correcte"
Architecte d'abord
Demande toujours 3 approches avec trade-offs avant d'implémenter. Tu choisis. Claude exécute. → Évite de se retrouver à mi-chemin dans la mauvaise direction.
06 — Configuration

Le CLAUDE.md — ta mémoire permanente

~/.claude/CLAUDE.md (global) + ./CLAUDE.md (projet)

Ce fichier est chargé automatiquement à chaque session. Tu n'as plus jamais à réexpliquer le contexte. Hiérarchie : global → projet racine → sous-dossier.
